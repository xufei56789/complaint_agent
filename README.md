# 工单智能分析 Agent

基于 LangGraph + 豆包（ByteDance Ark）大模型，对 12345 市民热线工单进行自动化智能分析，输出四要素提取、四级分类、群诉识别三份结构化结果，并生成格式化 Excel 报告。

## 功能概览

| 功能 | 说明 |
|------|------|
| **四要素提取** | 从工单文本中识别时间、地点、人物、事件 |
| **四级内容分类** | 按 12345 市民热线标准体系进行一/二/三/四级分类 |
| **群诉识别** | 聚合同点位、同问题的多条工单，识别潜在群体性投诉 |
| **Excel 报告** | 生成三个标签页的格式化结果文件 |

## 工作流拓扑

```
加载数据 → 四要素提取 → 四级分类 → 群诉检测 → 生成 Excel
```

## 环境要求

- Python 3.12+
- 豆包 API Key（ByteDance Ark 平台）

## 安装

### 方式一：pip（推荐）

```bash
# 克隆项目
git clone <repo_url>
cd complaint_agent

# 创建虚拟环境
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt
```

## 配置

### 1. 设置 API Key

```bash
export ARK_API_KEY="你的豆包 API Key"
```

或直接修改 `config/settings.py`：

```python
ARK_API_KEY = "你的豆包 API Key"
```

### 2. 准备输入数据

将工单数据放到 `data/测试数据.xlsx`，文件需包含以下列：

| 列名 | 说明 |
|------|------|
| `工单编号` | 工单唯一标识 |
| `主要内容` | 工单文本内容 |

### 3. 其他可选配置（`config/settings.py`）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `LLM_MODEL` | `doubao-seed-2-0-lite-260428` | 使用的模型名 |
| `LLM_TEMPERATURE` | `0.0` | 温度（分类任务建议保持 0） |
| `BATCH_SIZE` | `10` | 每批并发调用 LLM 的工单数 |
| `GROUP_MIN_COUNT` | `2` | 群诉判定的最小工单数阈值 |

## 输出说明

结果文件保存在 `results/结果.xlsx`，包含三个标签页：

**Sheet 1 - 四要素**

| 工单编号 | 时间 | 地点 | 人物 | 事件 |
|----------|------|------|------|------|

**Sheet 2 - 四级分类**

| 工单编号 | 一级分类 | 二级分类 | 三级分类 | 四级分类 |
|----------|----------|----------|----------|----------|

一级分类范围：城市管理 / 住房建设 / 市场监管 / 教育卫生 / 社会民生

**Sheet 3 - 群诉**

| 群诉编号 | 工单编号 | 涉及点位 | 共性问题 | 涉及工单数 | 事件摘要 |
|----------|----------|----------|----------|------------|----------|

群诉编号前缀：`T1-xxx`（位置+问题联合聚合）、`T2-xxx`（小区名聚合）

## 项目结构

```
complaint_agent/
├── main.py                 # 程序入口
├── config/
│   ├── settings.py         # API Key、路径、批处理参数
│   └── classification.py   # 四级分类关键词体系 + LLM system prompt
├── graph/
│   ├── state.py            # AgentState 状态定义
│   ├── builder.py          # LangGraph 工作流组装
│   └── nodes.py            # 5 个节点函数（LLM 调用逻辑）
├── tools/
│   ├── extractor.py        # 四要素规则提取 + 关键词分类
│   └── group_detector.py   # 群诉识别工具函数
├── output/
│   └── excel_writer.py     # Excel 格式化输出
├── data/
│   └── 测试数据.xlsx        # 输入数据（需自行准备）
├── results/
│   └── 结果.xlsx           # 输出结果（运行后生成）
└── requirements.txt
```
